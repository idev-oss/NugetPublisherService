## Описание

NugetPublisherService - это автоматизированный сервис для публикации NuGet-пакетов в приватном репозитории GitLab. Сервис работает в фоновом режиме, регулярно сканирует указанную директорию на наличие новых пакетов и выполняет их публикацию в репозиторий GitLab NuGet.

## Ключевые возможности

- **Быстрое адресное сканирование**: Прямой обход известной структуры папок (`год → дата → leaf`) без рекурсии по всему дереву шары; обрабатывается только окно последних дней
- **Локальный кэш на SQLite**: Уже обработанные пакеты пропускаются без чтения `.nupkg` и без обращения к GitLab; дедуп переживает рестарт сервиса
- **Интеллектуальное расписание**: Разные интервалы сканирования в рабочее и нерабочее время (настраиваются)
- **Проверка дубликатов**: Предотвращение повторных публикаций уже существующих пакетов
- **Email-уведомления**: Автоматическая отправка отчетов с тремя статусами (Успешно / Ошибка / Пропущено в DryRun)
- **Уведомления об ошибках без спама**: Письмо об ошибке публикации отправляется один раз по достижении порога `FailureAlertThreshold`; администраторам (`Smtp.Admin`) — с деталями, обычным получателям — просьба обратиться к администратору
- **Обработка отказа авторизации**: При HTTP 401/403 от GitLab сервис не ретраит впустую — уведомляет только администратора и корректно останавливается (после исправления токена службу нужно запустить снова)
- **Режим "Dry Run"**: Возможность тестировать работу сервиса без фактической публикации пакетов

## Принцип работы

1. Сервис строит пути напрямую по структуре `BasePath\{год}\{дата}\{leaf}` и сканирует только папки дат в окне `LookbackDays`
2. Для каждого `.nupkg` сверяется с локальным SQLite-кэшем; уже опубликованные пропускаются
3. Незнакомые кэшу пакеты разово проверяются в GitLab (засев кэша), новые публикуются через NuGet API
4. Отправляет email-отчет о результатах публикации

## Конфигурация

Сервис настраивается через файл `appsettings.json`. Готовый шаблон с комментариями — [appsettings.example.json](NugetPublisherService/appsettings.example.json).

```jsonc
{
  "Scan": {
    "BasePath": "\\\\nas\\app\\update",   // корневая папка (сетевая шара)
    "YearFolderFormat": "_yyyy",            // имя папки года: _2026
    "DateFolderFormat": "yyyy-MM-dd",       // имя папки даты: 2026-05-18
    "LeafRelativePath": "Refactor\\NugetSource", // подпуть до .nupkg
    "LookbackDays": 60,                     // окно сканирования (прошлый+текущий месяц)
    "IncludePreviousYearFolder": true,      // учитывать прошлый год на стыке (январь)
    "ScanIntervalMinutes": 10,              // интервал в рабочие часы
    "WorkingHourStart": 9,                  // начало рабочих часов
    "WorkingHourEnd": 21,                   // конец рабочих часов
    "OffHoursIntervalHours": 2              // интервал вне рабочих часов и в выходные
  },
  "GitLab": {
    "BaseUrl": "http://git.example.local/api/v4",
    "ProjectId": 11,
    "PrivateToken": "токен_доступа"
  },
  "Smtp": {
    "Server": "smtp.example.com",
    "Port": 465,
    "UseSsl": true,
    "Username": "username",
    "Password": "password",
    "From": "sender@example.com",
    "To": ["recipient1@example.com", "recipient2@example.com"],
    "Admin": ["admin@example.com"]          // получатели подробностей об ошибках
  },
  "State": {
    "DatabasePath": "state.db"              // файл локального SQLite-кэша (рядом с .exe)
  },
  "DryRun": false,
  "FailureAlertThreshold": 5                 // письмо об ошибке после N неудачных циклов (один раз)
}
```

> Итоговый путь поиска: `BasePath\_2026\2026-05-18\Refactor\NugetSource\*.nupkg`.
> Структура папок настраивается форматами `YearFolderFormat` / `DateFolderFormat` / `LeafRelativePath`.

Конфигурация валидируется при старте (fail-fast): при некорректных значениях сервис не запустится
и выведет понятную ошибку. Существование сетевой папки `BasePath` проверяется в рантайме
(временная недоступность шары не валит сервис).

## Установка

1. Клонируйте репозиторий
2. Настройте файл `appsettings.json` согласно вашим параметрам
3. Соберите и запустите сервис

**Служба Windows:** Для продакшн развертывания приложение может быть установлено как служба Windows с использованием предоставленного скрипта [install_service.ps1](NugetPublisherService/install_service.ps1).

## Требования

- .NET 10.0
- Доступ к репозиторию GitLab с поддержкой NuGet package registry
- SMTP сервер для email-уведомлений (опционально)
